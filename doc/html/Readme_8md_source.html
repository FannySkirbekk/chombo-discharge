4 Building the documentation
5 ==========================
6 
7 Before proceeding further, please install doxygen and build the code documentation. This requires that doxygen is installed on your computer, but it does not require a working installation of the code. To build the documentation, please type
8 
9  doxygen doxyfile
10 
11 in your shell (from the main folder). This will compile the documentation of the code in which you will find installation instructions. After building the documentation, please proceed ../chombo-streamer/doc/doxygen/html/index.html and navigate to installation instructions.

14 Installation instructions
15 =========================
16 
17 Prerequisites
18 ----------------------
19 
20 The numerical code is written in C++ using the [MFChombo](https://commons.lbl.gov/display/chombo/Chombo+-+Software+for+Adaptive+Solutions+of+Partial+Differential+Equations) library and can be compiled on UNIX systems (both workstations and clusters).
21 Compilation requires the following prerequisites to be in place:
22 
23 1. A C++ compiler (e.g. g++, icpc)
24 2. A Fortran compiler (e.g. gfortran, ifort)
25 3. An MPI compiler (e.g. OpenMPI)
26 4. A working HDF5 installations, both parallel and serial.
27 
28 Compilers can be installed straightforwardly on UNIX systems; for users of Ubuntu-like systems we recommend that you install g++ (and gcc), gfortran, and OpenMPI.
29 
30 
31 HDF5 installation
32 -----------------
33 
34 Before attempting to install HDF5, please make sure that C++, Fortran, and MPI compilers are in place.
35 
36 HDF5 (Hierarchical Data Format) is a portable, binary data file with parallel support and is used to reduce I/O bottlenecks in simulations. HDF5 can be obtained from [here](https://support.hdfgroup.org/HDF5/).
37 
38 While it is possible to compile our code for serial execution only, we generally recommend that the user installs both serial and parallel versions of HDF5. The two versions should *not* be installed in the same directory.
39 For versions of HDF5 newer than 1.6, we require that the user configures HDF5 with the 1.6 API version.
40 
41 1. Compile and install zlib (a compression library). zlib can be obtained through
42 
43  sudo apt-get install zlib1g-dev
44 
45 2. Download and install HDF5 for *serial execution*.
46 
47  ./configure --prefix=/usr/local/hdf5 --enable-production --enable-cxx --enable-fortran --with-default-api-version=v16
48  make
49  make install
50 
51  This will install a serial version of HDF5 in /usr/local/hdf5 (if you want to install
52 
53 3. Install HDF5 for *parallel execution*
54 
55  ./configure --prefix=/usr/local/phdf5 --enable-production --enable-fortran --with-default-api-version=v16 --enable-parallel
56  make
57  make install
58 
59  This will install a parallel version of HDF5 in /usr/local/phdf5

62 Building the source code
63 ------------------------
64 
65 Once sequential and parallel versions of HDF5 are installed, the source code may be built. The source is built by inheriting a set of makefiles from Chombo. By default, a version of Chombo-3.2 is shipped with this code.
66 To build the source code, you must ensure that the makefile finds the Chombo library by modifying the GNUmakefile which lies at the top directory in the code. Modify this file so that `CHOMBO_HOME` points to the correct location (a full path specification may be necessary).
67 For example, if the code is in the home folder of a user `exampleUser`, `CHOMBO_HOME` should be
68 
69  CHOMBO_HOME := /home/exampleUser/FluidAMRStreamer/Chombo-3.2/lib
70 
71 Specification of default settings for Chombo are necessary for compiling both the Chombo library and the source code. You should modify Make.defs.local which lies in Chombo-3.2/lib/mk/ for specification of compilers and path to the serial and parallel HDF5 installations. If you installed HDF5 in /usr/local/hdf5 and /usr/local/phdf5, the default settings in Make.defs.local are (probably) sufficient.
72 
73 The streamer code can be compiled for two-dimensional and three-dimensional executation on Cartesian grids (default is 2D). To compile the code, it should be sufficient to navigate to one of the plasma models and then execute
74 
75  make "model"
76 
77 where "model" is replaced by the model name. If you want to silence GNUmake during the compilation stage, include a flag -s. I.e.
78 
79  make -s "model"
80 
81 This will inherit a set of makefiles: Chombo source codes are compiled first and the streamer code is compiled second. To execute for three-dimensional execution, either make appropriate changes to Make.defs.local (DIM=3), or run
82 
83  make DIM=3 "model"
84 
85 By default, the executable will be named according to the specifications in Make.defs.local. For example, if the code was compiled for optimized three-dimensional execution using MPI (mpiCC as compiler) gfortran on a Linux system, the executable is named
86 
87  plasma3d.Linux.64.mpiCC.gfortran.OPT.MPI.ex
88 
89 The code has compiled successfully if this file is created.

91 Compilation on vilje
92 ====================================
93 
94 To compile the system on the *vilje* supercomputer, you need to load compilers and libraries before compiling the code. We recommend that you use intel compilers by loading
95 
96  module load intelcomp/15.0.1
97 
98 which will load Intel compilers for Fortran and C/C++ (version 15.0.1). MPI libraries are loaded by
99 
100  module load mpt/2.10
101 
102 and HDF5 is loaded by
103 
104  module load hdf5/1.8.14
105 
106 It is likely possible to load newer versions of compilers, MPI, and HDF5 (but we haven't tried). Once the libraries are loaded, the code may be compiled as usual. It will be necessary to update CHOMBO_HOME and supply the library path to the Chombo makefiles.
107 CHOMBO_HOME is simply updated to the folder in which Chombo is installed. HDF5 paths are found by typing
108 
109  module disp hdf5/1.8.14
110 
111 which will display the library paths to HDF5. The typical output is
112 
113  /sw/sdev/modulefiles/hdf5/1.8.14:
114 
115  module-whatis general purpose library and file format for storing scientific data
116  prereq intelcomp/15.0.1
117  prereq mpt/2.10
118  prepend-path PATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/bin
119  prepend-path CPATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/include
120  prepend-path FPATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/include
121  prepend-path LIBRARY_PATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/lib
122  prepend-path LD_LIBRARY_PATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/lib
123  prepend-path PYTHONPATH /sw/sdev/Modules/hdf5/hdf5-1.8.14/lib/python2.7/site-packages
124  setenv HDF5_DIR /sw/sdev/Modules/hdf5/hdf5-1.8.14
125 
126  Loads HDF5 version 1.8.14
127 
128 To use Intel's Fortran and C/C++ compilers, please specify
129 
130  FC = ifort
131  CXX = icpc
132 
133 in Make.defs.local. Likewise, update Make.defs.local so that it finds the HDF5 libraries and include files. For this version of HDF5, they are located in
134 
135  /sw/sdev/Modules/hdf5/hdf5-1.8.14/include
136  /sw/sdev/Modules/hdf5/hdf5-1.8.14/lib
137 
138 With these paths, the appropriate flags in Make.defs.local are
139 
140  HDFINCFLAGS = -I/sw/sdev/Modules/hdf5/hdf5-1.8.14/include
141  HDFLIBFLAGS = -L/sw/sdev/Modules/hdf5/hdf5-1.8.14/lib -lhdf5 -lz
142 
143  HDFMPIINCFLAGS = -I/sw/sdev/Modules/hdf5/hdf5-1.8.14/include
144  HDFMPILIBFLAGS = -L/sw/sdev/Modules/hdf5/hdf5-1.8.14/lib -lhdf5 -lz

146 Compilation on fram
147 ====================================
148 To compile the system on the *fram* supercomputer, you need to load compilers and libraries before compiling the code. The intel compiler toochain is loaded by
149 
150  module load intel/2017a
151 
152 which will load Intel compilers for Fortran and C/C++, as well as Intel MPI. The code may be compiled as usual. It will be necessary to update CHOMBO_HOME and supply the library path to the Chombo makefiles.
153 CHOMBO_HOME is simply updated to the folder in which Chombo is installed. HDF5 paths are found by typing
154 
155  module disp HDF5/1.10.1-intel-2017a
156 
157 which will display the library paths to HDF5.
158 
159 To use Intel's Fortran and C/C++ compilers, please specify
160 
161  MPICXX = mpiicpc
162  FC = ifort
163  CXX = icpc
164 
165 in Make.defs.local. Likewise, update Make.defs.local so that it finds the HDF5 libraries and include files.
166 
167 We also recommend that you compile with -xCORE-AVX2 (put it in cxxoptflags and foptflags) to enable advanced vector extensions.
